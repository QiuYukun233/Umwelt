## 1. 动机

当前 HTML 的 2D 节点+连线编辑器已到表达力极限,而工坊是玩家投入最多时间的核心界面 —— 对应架构文档循环 A 的"修复"环节、循环 B 的"设计"环节。

把工坊迁到 Bevy,并把基质定为**网格 + 分层 + 垂直 via + 横截面读取**。研究报告论证:网格化布线同时是(a)Zachtronics 全系验证的"可读性"约定 —— 直线布线、占地/via 计成本、可视化 diff,(b)对真实昆虫神经组织的诚实抽象 —— 昆虫神经毡本就组织成离散板层(视叶 medulla M1–M10、蘑菇体)、带视网膜拓扑柱与垂直投射。

HTML 在过渡期继续承担主世界沙盒(蚂蚁涌现观察场),工坊导出的模块通过 JSON 装回 HTML 蚂蚁身上跑。长期目标整个游戏迁 Bevy。本设计只覆盖"工坊侧"。

## 2. 核心范式 — 网格化解剖空间

Bevy 工坊里,3D 空间是器官本身,组织成**堆叠的 2D 网格层**:

- 每一层是一个 2D 网格,对应一个**解剖板层**(真实昆虫神经组织正是这样分层的)。
- 每个网格格子可容纳一个神经元、一个 via,或留空。
- 轴突沿格子做**曼哈顿布线**(只走正交方向,无曲线、无斜线)。
- **via** 是唯一的跨层原语:轴突要换层,必须在某格放一个 via 垂直穿过。
- 读取靠**任意轴向横截面**:玩家自上而下逐层写,但可以在任意轴对齐的面上切片来读 —— 这是分析 3D 布线的primitive(对应 VLSI 截面视图、连接组学连续切片)。

空间位置 = 生物位置:神经元在它解剖学应在的层与格上。这个范式服从架构文档"诚实于神经结构的逻辑"。

**约束创作、不约束动力学。** 网格约束的是*布线的写法*(为了可读性与可 diff),不约束*电路的动力学*:分级信号、递归、侧抑制、神经调制都在同一网格基质里平权存在(见 §3)。

## 3. 自由度 — 网格上的全 DIY

工坊不强迫玩家做"一根触须"。系统**只声明 I/O 契约**(关卡给定的输入受体类型、输出端口)。

工坊把器官放进一个**无限可扩的 3D 网格魔方** —— 没有空间墙。约束玩家的不是边界,是 §4 的真实生理代价。

玩家自由决定:

- **受体类型与物理采样位置** —— 从关卡受体调色板里选,放在器官凸包外表面的任意位置。受体的物理位置决定它采样化学场的哪一点(§5)。
- **神经元数量,及每个的层 / 格位置**。
- **每根轴突的逐格布线路径、via 放置、轴突粗细 d**(§4.1)。
- **可塑性区域** —— 哪些连接可塑、绑哪个 modulator。

**硬约束**:轴突只能走曼哈顿网格;跨层必须用 via。
**预算约束**:体积、代谢功率、传导延迟、信号衰减(§4)。

**递归 / 侧抑制 / 调制是一等公民**:它们和前馈连接走同一网格、同一布线规则,无额外摩擦。这是刻意的不变量 —— 防止"分层网格"被误读、被实现成前馈深度网络(研究报告反复警告这个失败模式:真实板层结构里递归与侧抑制无处不在)。

**节点类型(5 种,不新增)**:sensor_on、inter_exc、inter_inh、modulator、motor —— 不变,只是现在它们住在网格格子里。

**哲学**:玩家可能造出完全不像真实蚂蚁器官的东西,只要它履行 I/O 契约 —— 合法且被鼓励。真实生物的形态本身是演化的解,不是设计的前提。

## 4. 物理约束(真实生物学)

工坊里的空间不是装饰。玩家"把这个神经元放远一点"这一个决定,会同时撞上多组物理后果。这些约束基于真实昆虫神经组织的物理事实,而非虚构的"代价滑块"。

**信号模型:分级,非脉冲。** 全系统用 0.0–1.0 连续分级信号,没有脉冲(spike)。这不是图省事 —— 它是下面三条约束能互相诚实的前提(见约束 3)。

### 4.1 三条物理约束

工坊把器官放进无限可扩的 3D 网格;没有空间墙。约束玩家的是三组真实生理代价。

**约束 1 · 传导延迟 ∝ 轴突路径长度。** 信号沿轴突传播速度有限,距离 = 真实时延。轴突路径 = 玩家逐格画出的曼哈顿布线(含跨层 via 段)。这条对分级和脉冲信号都成立。编译进 HTML 运行时即 `edge.delay_ms`(已实现)。

**约束 2 · 代谢预算。** 器官有一个**可持续功率上限**(每秒能烧多少 pJ/s),与 episode 时长无关。代谢成本是连续的瞬时功率,三项都用 pJ/s,量纲一致、彼此可加:

1. 每神经元维持静息电位的**静息功率**;
2. 每神经元的**活动功率** ∝ 其当前分级激活(**瞬时,不对时间积分** —— 时间积分得到的是能量,会把代谢和运行时长绑死,而时长属于延迟 / cycles 那条轴,违反 §4.3 的正交性);
3. 每突触的**维持功率**。

*(注:日后若要"能量耗尽 / 疲劳"机制,那是另设一条随消耗衰减的能量储备,独立 feature,不与这条功率预算混。)*

*(早先草稿写的"每次 spike 额外代谢成本"已删 —— 系统无 spike,按脉冲计价与无脉冲模型自相矛盾。)*

**约束 3 · 信号随轴突距离衰减。** 分级信号沿轴突会电紧张式衰减(electrotonic decay):远端振幅 = 输入振幅 × exp(−路径长 / λ),λ 是轴突长度常数。**这条只对分级信号诚实** —— 脉冲是再生的、振幅不随距离变,给脉冲加衰减就是虚构一个不存在的物理。反过来说,衰减正是分级信号在真实生物里"必须局部"的原因,也是本系统选择无脉冲模型后必须诚实承担的后果。

**轴突粗细是三条约束的共同杠杆。** 玩家给每根轴突设粗细 d。d 一变,三件事同时动 —— 因为它们是同一物理的不同侧面:

- 长度常数 λ ∝ √d(更粗 → λ 更大 → 约束 3 衰减更小);
- 传导速度 ∝ √d(更粗 → 更快 → 约束 1 延迟更小);
- 体积 ∝ d² × 路径长 → 代谢维持随体积上升(约束 2)。

速度快和抗衰减不是两个巧合,是同一根 √d 的两面:传导速度 v ∝ λ/τ_m,而膜时间常数 τ_m 与 d 近乎无关,所以速度跟着 λ 走。`constants/biology.rs` 的相关常数注释必须点明这个机理。粗轴突 = 又快又"韧"但占体积、耗代谢;细轴突 = 省体积代谢但又慢又易衰减。

**几何约束。** 体积预算:神经元胞体 / 轴突 / 突触都占体积;总体积 = **Σ(每层各自足迹 × 层高 h)** —— 逐层求和,不是"合并足迹 × 总层高"的棱柱(后者让只占某层一角的稀疏层按全层合并的大足迹付费,不诚实;逐层求和也不更难算)。总体积有上限。感受器必须暴露:化学 / 机械感受器细胞必须落在器官凸包的外表面。双侧空间分辨率:两个化学感受器之间的空间基线越小,解方向的信噪比越差 —— 这是仿真层面的物理后果,不是数值惩罚。

*(原草稿的一次性"建造成本"已删:它 ∝ 神经材料总量,和体积预算近共线 —— 两条等额上限其实约束同一件事,违反正交性。体积预算独自承担"别造太多"。)*

**via 不单独计价。** via 是轴突路径的一段;路径长一旦进入约束 1/2/3 的计价,via 已被算过,不再加一个冗余的 via 计数。

### 4.2 数值的真实性策略

"量级正确":

1. 常数用真实物理单位(μm³、pJ/s、m/s、μm 的 λ),不用无单位"能量点"。
2. MVP 数值取在真实昆虫数据一个量级之内,允许粗略估算。
3. 常数集中在 `constants/biology.rs`,每个附文献来源注释。
4. 后期精校时改数字,不改单位制。

**关于来源(见 §8 #9)**:研究报告提供的是*结构*和*哪些常数是诚实的*,不是一张数值表。λ / 传导速度 / pJ 的具体量级报告基本没直接给 —— 这些多数会是 §4.2 式估算,照常在常数注释里注明"估算"及最接近的文献依据。

### 4.3 物理后果的玩家可见性

工坊 UI 实时显示玩家这套设计的当前消耗 —— 体积占用 / 总体积、代谢功率 / 总代谢、最长信号路径延迟。**这几个数永远分开显示,绝不合成单一"效率分"**:它们是正交的物理量,合并会抹掉玩家要做的权衡。

**每关一个静态 par。** 每个关卡附一个 par —— 关卡作者已知解的成本基线(如"神经元数 par = 7")。工坊显示"你用了 N,par 是 P"。不需要分布、不需要社区,就一个数,给单人优化一个靶子;关卡作者验证关卡时本就得自己解出来,par 几乎是免费副产物。社区式的成本分布直方图(多个解的柱状对比)推迟到有社区之后。

## 5. 关卡形态:两段式测试

### 5.1 测试结构

**Stage 1 · 静态探针。** 系统在玩家器官周围摆一个静态场快照(3D、固定)。玩家的受体按其物理位置真的采样这个场。电路跑预定时长,系统记录输出波形并对照规范带。特点:决定性、可重放、可调试;受体位置真的有意义;一关含 N 个探针,全过才算 Stage 1 完成。

**Stage 2 · 动态竞技场。** Stage 1 通过后解锁。系统把玩家器官装到一只测试蚂蚁,放进小型 3D 测试环境(有化学源、动态演化场)。验真实行为("5 秒内到达 ChemA 源"等)。多 trial,成功率 ≥ 阈值。

**两阶段都通过 → 模块进入玩家的模块库(导出)。**

### 5.2 测试场的表示:可扩展标量场

受体在自己的物理位置采样一个**静态标量场**(每种感知通道一个场:ChemA 场、ChemB 场、…,以及后续关卡的几何场)。一个场由若干 **contributor** 组合而成。contributor 分类型,**组合规则随类型不同**:

- **化学点源**(chemical point source):位置 + 强度 + 空间衰减剖面。同种化学物的多个点源**按和(Σ)组合**。Stage 2 里点源额外带扩散 / 挥发的时间演化参数;Stage 1 里点源是静态的。
- **几何距离场**(geometry distance field):由障碍几何派生,场值 = 到最近障碍的距离(或其函数)。**按 min 组合**,不是 Σ。MVP 不使用 —— 仅为后续避障关在 schema 里留口。

场之上还有一层 **phase / 时序外壳**:一次测试由若干 phase 组成,每个 phase 有自己的场配置 + 时长 + 判定。**MVP 只有一个 phase。** 这一层为后续可塑性关(配对训练 phase → 测试 phase + 定时奖励 / 调制)留口。

**一套仿真器两种调用。** 化学场仿真器(Bevy 侧子系统 B)只有这一种场表示。Stage 1 = 场是静态快照(contributor 不演化);Stage 2 = 场按 contributor 的时间参数演化。Stage 1 不是另一套系统,是同一套的静态调用。

> **schema 留口的理由(§8 #7)**:I/O 契约、两段式结构、逐格布线都很稳,容得下后续关卡;脆的是场表示。避障关需要不可叠加的几何距离场(min,非 Σ),可塑性关需要时序 —— 把"多 contributor 类型 + phase 外壳"从第一天就写进 schema,避免后面迁 schema。两者都不给 MVP 加功能,只留口。

### 5.3 第一关:化学定向器官

**关卡 ID**:`chemotaxis-l1`
**输入受体调色板**:ChemA 化学感受器、ChemB 化学感受器。
**输出端口**:`turn_L`、`turn_R`、`forward`。
**Stage 1 静态探针**(草案 —— 每个探针 = 一个 phase + 一组静态化学点源 + 期望输出带):

- probe-1:一个左侧 ChemA 点源,无 ChemB → 期望 `turn_L` > 0.5(200ms 内)。
- probe-2:一个右侧 ChemA 点源 → 期望 `turn_R` > 0.5。
- probe-3:无任何源 → 期望 |turn_L − turn_R| < 0.1。
- probe-4:一个左侧 ChemB 点源(危险物)→ 期望 `turn_R` > 0.5(避开)。
- probe-5:同侧的 ChemA + ChemB 点源 → 期望某种约定行为(关卡设计点)。

**Stage 2 动态场景**(草案):trial-1 单 ChemA 源任意方位 → 5 秒内距源 < 阈值;trial-2 单 ChemB 源 → 5 秒内远离;trial-3 同时存在 ChemA / ChemB 源 → 趋 A 避 B。

具体阈值与时长在实现阶段调。

## 6. MVP 作用域

**作用域:裸关卡 + 模块导出。**

**包含**:

- Bevy 网格工坊运行时:堆叠层、网格放置神经元 / via、逐格曼哈顿布线、轴突粗细编辑、可塑性指定、横截面读取、相机控制。
- 化学场仿真器(子系统 B,§5.2 的可扩展标量场,MVP 只用化学点源 + 单 phase)。
- 第一关 `chemotaxis-l1` 完整可玩:加载 → 设计 → Stage 1 → Stage 2 → 通过 → 导出模块 JSON。
- 物理约束实时显示 + par(§4.3)。
- 模块 JSON 格式(§7),HTML 侧可加载。

**不包含**(post-MVP):

- 成本分布直方图(社区功能)、关卡选择菜单 / 模块库 UI / 工坊壳。
- 修复关 —— 软 trigger ≈ MVP 刚完(§8)。
- onboarding 脚手架 —— gated-on-first-playtest(§8)。
- 元件库系统(架构文档循环 B);主世界(仍由 HTML 承担);化学场仿真器的 WASM 共用。

MVP 验证单一问题:**「网格解剖工坊 + 真实物理 + 两段式测试」这套核心循环是否好玩**。MVP 第一个用户是作者本人,不是陌生玩家 —— 故第一关不带 onboarding 脚手架,要验的是 authoring 流程 + 动力学。

## 7. 工坊 ↔ HTML 主世界 的合约

### 7.1 导出格式

通过关卡后,工坊导出一个 JSON 模块(`umwelt-module-v1`,沿用已实现的 HTML 加载器):

```jsonc
{
  "schema": "umwelt-module-v1",
  "level_id": "chemotaxis-l1",
  "compiled_at": "<iso8601>",
  "receptors": [ { "id": "r0", "type": "chem_A", "body_pos": [x,y,z], ... } ],
  "graph": { /* 编译后的 NeuralGraph 序列化,edge 带 delay_ms 与 attenuation */ },
  "meta": { "volume_used_um3": ..., "metabolic_pj_s": ..., "max_path_delay_ms": ..., "grid": {...} }
}
```

### 7.2 编译策略:A-lite

工坊里的真实物理在导出时**部分编译进 HTML 运行时**:

- **传导延迟**:每条轴突的曼哈顿路径长度按传导速度算成 `delay_ms`,作为 `edge.delay_ms` 进入导出图。HTML evaluator 已用环形缓冲实现(schema v10,已上线)。
- **信号衰减**:每条轴突的路径长度 + λ 算出一个衰减因子,作为每边一个乘子进入导出图。**这是与 delay 同类的"距离 → 动力学"诚实链**,且实现极廉(每边一个乘子)—— 故 A-lite 把它和 delay 一同编译进 HTML。⚠️ *本条是写 spec 时的新增决定(§4 约束 3 是本轮新加的),需在 spec review 时确认。*
- **体积、代谢功率**:仅在工坊里作"关卡通过门槛"履行。装载到 HTML 后不再影响运行时,以元数据形式跟着模块走,供 HTML UI 显示。

### 7.3 为什么是 A-lite

游戏长期愿景是全 Bevy 化,届时 3D 物理在运行时完整存在。HTML 是过渡期临时主世界,为它加的运行时逻辑最终都会随 HTML 被替换。但 **"距离 → 延迟"和"距离 → 衰减"这两根诚实链不能丢** —— 否则玩家在工坊里调的空间设计(近放 / 远放、粗轴突 / 细轴突)装到主世界后毫无区别,工坊的空间设计在主世界就失声。体积 / 代谢不一样:它们的作用在工坊设计阶段已经发生,装载后玩家不再编辑电路,运行时只是死数据 —— HTML 阶段不为它们写运行时逻辑是合理取舍。

### 7.4 HTML 侧的局部改动

- `edge.delay_ms` + 环形缓冲 + 存档 schema v10 + "装载模块"入口 —— **已实现并上线**(2026-05-21 / 05-22)。
- **新增待办**:`edge` 再加一个 attenuation 衰减因子(每边一个乘子,evaluator 里乘进去)。与 delay_ms 同类的小改动,随首个 Bevy 计划一并排期。

主世界其他部分(化学场、蚂蚁运动学、可塑性更新规则)不动。

## 8. 开放问题的收敛

原 §8 的 10 条开放问题,本轮 brainstorming 的收敛结果:

1. **3D 编辑器交互模型** —— 由网格基质解决(§2):逐层网格放置 + 逐格曼哈顿布线 + 显式 via + 横截面读取。鼠标 / 键盘的具体操作方案留实现期 UX 细化,但模型已定。
2. **轴突几何粒度** —— 解决:逐格曼哈顿走线 + 显式 via,无曲线 / 贝塞尔。粗细 d 是每根轴突一个参数(§4.1)。
3. **器官包络** —— 解决:不雕壳。无限网格魔方,器官范围是细胞位置的涌现结果;凸包足迹(逐层 Σ)是成本指标,不是墙(§3 / §4)。
4. **Stage 1 静态场构造** —— 解决:可扩展标量场,化学点源叠加,与 Stage 2 同一表示(§5.2)。
5. **通过 / 失败诊断反馈** —— 解决(MVP 档):测试报告指出①哪个探针 / trial 失败,②该探针的输出波形 vs 期望带的对照。成本侧见 §4.3(自己的数 + par)。报告那套对比直方图是社区功能,推迟;差异分析 / 智能提示不做 —— MVP 第一个用户是作者,看波形对照足够。
6. **模块版本演化** —— 解决:导出模块是纯文本、可 diff 的(社区前提)。重做某关 → 重新导出 → 覆盖。不做版本树。
7. **第二 / 三关形态预想** —— 预想(非硬决定,只为约束关卡格式):下一批关大概是机械感受器避障、可塑性学习关;报告押注的"10 只蚂蚁涌现路径"是远端目标关。**契约 / 两段式测试结构容得下后续关;Stage 1 场 schema 已按可扩展设计(多 contributor 类型 + phase 外壳,§5.2)以容纳避障关的几何距离场与可塑性关的时序。**
8. **Bevy / HTML 代码组织** —— 已决定(2026-05-21):Bevy 独立 git 仓库。
9. **真实生物学常数源** —— 解决:以研究报告(`docs/compass_artifact_*.md`)引用的文献为常数*结构*基底。⚠️ 注意:报告给的是结构与"哪些常数诚实"(板层结构、Dale 定律、分级信号、可塑性定位、布线经济 —— Cherniak 1994、Bullmore & Sporns 2012 等),**不是一张数值表**;λ / 速度 / pJ 的具体量级多数仍是 §4.2 式估算。`biology.rs` 每常数附文献注释,估算处注明。
10. **许可证** —— 仍开放(非阻塞):Bevy 是 MIT/Apache 双许可;游戏本体许可证策略待定,首个 Bevy 计划落地前定即可。

**本轮专门敲定的三点:**

- **par** —— 每关一个静态 par 成本基线(§4.3)。单人优化靶子,不需社区。
- **参考电路 —— 拆成两件独立追踪,不捆绑推迟:**
  - **修复关**(给坏电路让玩家修,一个谜题品类):软 trigger —— "MVP 后、有第一个已验证参考电路即可开"(≈ MVP 刚完)。修复关是最便宜的内容(lesion 一个已验证电路即成,MVP 本身就产出已验证电路),又是"修着修着就懂了"那个核心教学回路的引擎 —— 不让它在计划里承诺最弱。
  - **onboarding 脚手架**(半透明范例电路,玩家边建边描摹,教学 affordance):**gated-on-first-playtest** —— trigger = "第一个外部 playtester 上手"。spec 明确标这个 trigger,**不许滑成无限期推迟**:否则冷开局 UX 崩会被误读成设计失败,拿到假信号。MVP 第一关本身不带脚手架(MVP 第一个用户是作者)。
- **via 不单独计价** —— via 是轴突路径的一段,路径长进 §4 约束 1/2/3 时已被算过;覆盖研究报告原本列的"via count"成本指标 —— 本系统的延迟 / 衰减模型让它冗余。

## 9. 设计宪法检查

按架构文档第 14 节附录:

1. **是否引入系统级功能语义?** 否。关卡 I/O 契约只用物理通道(ChemA 受体、turn_L 端口),无"避障检测器"式功能命名。参考电路是*形状*,不带含义标注。
2. **是否基于虚构的生物学机制?** 否。网格分层 = 真实板层结构(视叶 M1–M10);分级信号、Dale 定律、可塑性定位都有文献支撑;尤其约束 3 的衰减只对分级信号成立 —— 模型刻意保持无脉冲以让三条约束互相诚实。
3. **是否把有 Umwelt 的主体和无 Umwelt 的工具混同?** 否。工坊是基础设施(工具),其中设计的器官是主体的一部分。
4. **是否用数值缩放替代结构性差异?** 否。后续关卡的差异来自结构性新元素 —— 新受体类型、几何距离场、可塑性时序 phase —— 不是缩放参数。
5. **是否给硬核或新手之一增加摩擦而另一方无收益?** 网格基质对新手摩擦更大(无骨架模板、要学布线)。onboarding 脚手架(gated-on-first-playtest)正是为此 —— 已作为有明确 trigger 的待办独立追踪,不许无限期推迟。
6. **是否让玩家设计能力之外的东西成为核心进度变量?** 否。通过关卡靠设计能力(电路能否履行 I/O 契约);par 给优化靶子,但它衡量的仍是设计能力,不是资源积累或时间投入。
