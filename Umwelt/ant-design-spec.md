# Umwelt — 蚂蚁硬件规格书

> 设计原则：诚实于神经结构的逻辑。不做语义层的东西，只做具体功能层的东西。
> 不追求一比一还原，但不做本质上不诚实的抽象。精简的方式是"少几个"，而非"合并成一个假概念"。

---

## 1. 身体结构

蚂蚁身体分三段：头部、胸部、腹部。六条腿连接在胸部。两根触角连接在头部前端，向左前方和右前方伸出。一对大颚在头部前下方。腹部末端有螫针/酸孔（化学物质释放口）。

运动方式：六足步行，抽象为朝向转动 + 前进速度。不模拟单腿步态。

---

## 2. 化学场（世界规则）

化学场中存在若干种独立的化学物质，每种化学物质有自己的扩散速率和衰减速率。

### 保留的化学物种

| 编号 | 来源 | 扩散特性 | 生物学对应 |
|------|------|----------|------------|
| ChemA | 环境（食物源持续释放） | 中等扩散，缓慢衰减 | 食物细菌代谢物 |
| ChemB | 蚂蚁腺体 α 分泌 | 低扩散，中等衰减 | 对应真实毒腺/Dufour腺的地面分泌物 |
| ChemC | 蚂蚁腺体 β 分泌 | 高扩散，快速衰减 | 对应真实大颚腺的挥发性分泌物 |
| ChemD | 环境（危险区域持续释放） | 中等扩散，缓慢衰减 | 有害化学物质 |

**设计说明：**

- 四种化学物质是精简后的数量。真实蚂蚁的化学世界有几十上百种分子。四种足够支撑核心玩法（觅食、路径标记、报警、避险），又不至于让新玩家面对过多通道。
- 系统不给任何化学物质贴语义标签。ChemB 是"路径信息素"还是"领地标记"，完全取决于蚂蚁的接线方式和群体行为的涌现结果。
- 每种化学物质的扩散速率和衰减速率是独立参数，可以调节。低扩散 + 慢衰减 = 地面持久痕迹；高扩散 + 快衰减 = 空气中的短暂气味云。

---

## 3. 传感器（输入硬件）

所有传感器输出分级信号 0.0–1.0，接入 sensor_on 节点。

### 3.1 触角化学感受（左右各一组）

每根触角上有多个化学感受通道，每个通道对应化学场中的一种化学物质。

| 传感器 | 位置 | 感受对象 | 采样方式 |
|--------|------|----------|----------|
| L_chem_A | 左触角 | ChemA 浓度 | 锥形采样，朝左前方 |
| L_chem_B | 左触角 | ChemB 浓度 | 锥形采样，朝左前方 |
| L_chem_C | 左触角 | ChemC 浓度 | 锥形采样，朝左前方 |
| L_chem_D | 左触角 | ChemD 浓度 | 锥形采样，朝左前方 |
| R_chem_A | 右触角 | ChemA 浓度 | 锥形采样，朝右前方 |
| R_chem_B | 右触角 | ChemB 浓度 | 锥形采样，朝右前方 |
| R_chem_C | 右触角 | ChemC 浓度 | 锥形采样，朝右前方 |
| R_chem_D | 右触角 | ChemD 浓度 | 锥形采样，朝右前方 |

共 8 个化学传感器通道。

**设计说明：**

- 锥形采样复用线虫版的 cone sampling 实现。左触角采样区覆盖身体前方偏左约 45° 的锥形区域，右触角镜像。
- 每个通道就是一根物理上独立的感觉毛（sensilla basiconica 多孔型）里的一个感觉神经元。不同化学物种激活不同的神经元，这是物理现实，不是抽象。
- 真实蚂蚁触角上有几百种嗅觉受体。四种是精简，但每一种的工作方式是诚实的。

### 3.2 触角机械感受（左右各一）

| 传感器 | 位置 | 感受对象 | 响应方式 |
|--------|------|----------|----------|
| L_touch | 左触角 | 物理接触 | 碰撞时输出 1.0，否则 0.0 |
| R_touch | 右触角 | 物理接触 | 碰撞时输出 1.0，否则 0.0 |

共 2 个机械传感器通道。

**设计说明：**

- 对应触角上的 sensilla trichodea 无孔型。毛被弯折时神经元放电。
- 检测范围是触角尖端前方的一小段距离（碰撞检测）。
- 真实触角还有温度、湿度感受器（sensilla coeloconica），精简掉。

### 3.3 口器接触化学感受

| 传感器 | 位置 | 感受对象 | 响应方式 |
|--------|------|----------|----------|
| mouth_chem | 口器 | 当前接触物体的化学属性 | 大颚合拢在物体上时，输出该物体的化学特征值 |

共 1 个通道。

**设计说明：**

- 对应口器上的 sensilla basiconica 单孔型（味觉）。需要物理接触。
- 和触角化学感受的区别：触角读的是化学场中漂浮的浓度（远程嗅觉），口器读的是碰到的实体物体的属性（接触味觉）。
- 输出值是连续的，不同物体类型产生不同的数值。系统不告诉玩家"这是食物"——只给一个数字，玩家需要自己搭回路来区分。

### 3.4 光感受

| 传感器 | 位置 | 感受对象 | 响应方式 |
|--------|------|----------|----------|
| light | 头部复眼 | 环境光照强度 | 分级信号 0.0（暗）– 1.0（亮） |

共 1 个通道。

**设计说明：**

- 大多数工蚁视力极差，这里只保留明暗一个通道。不做方向、不做图像。
- 可用于趋光/避光行为。极度精简但诚实——就是一个光敏细胞。

### 3.5 体内状态感受

| 传感器 | 位置 | 感受对象 | 响应方式 |
|--------|------|----------|----------|
| energy | 体内 | 能量储备 | 1.0（饱）→ 0.0（饿），随时间下降，进食后恢复 |
| damage | 体内 | 受到伤害 | 受到 ChemD 伤害时输出高信号，否则 0.0 |

共 2 个通道。

**设计说明：**

- energy 不是"饥饿感"这个语义概念。它是一个物理量——体内糖原储备的水平，随代谢消耗持续下降，进食后回升。
- damage 是伤害感受器（nociceptor），对身体组织损伤产生响应的感觉神经元。不是"疼痛"这个主观体验。

### 传感器总计

| 类别 | 数量 |
|------|------|
| 触角化学感受 | 8（左4 + 右4） |
| 触角机械感受 | 2（左1 + 右1） |
| 口器味觉 | 1 |
| 光感受 | 1 |
| 体内状态 | 2 |
| **合计** | **14** |

## 4. 执行器（输出硬件）

所有执行器由 motor 节点驱动，接受 0.0–1.0 的信号。

### 4.1 运动

| 执行器 | 功能 | 信号含义 |
|--------|------|----------|
| motor_forward | 前进 | 信号越高，前进速度越快 |
| motor_turn_L | 左转 | 信号越高，左转越急 |
| motor_turn_R | 右转 | 信号越高，右转越急 |

共 3 个运动通道。

**设计说明：**

- 六足步态不模拟。运动抽象为朝向+速度。这是对线虫版 dorsal-bend/ventral-bend 的替换，不是语义抽象——蚂蚁的步行确实在宏观上可以描述为转向+前进。
- 没有后退。蚂蚁可以后退，但极少使用，精简掉。

### 4.2 腺体（化学物质释放）

| 执行器 | 位置 | 释放物质 | 释放方式 |
|--------|------|----------|----------|
| gland_α | 腹部末端 | ChemB | 腹部接触地面时，向当前位置的化学场写入 ChemB |
| gland_β | 头部大颚 | ChemC | 大颚张开时，向当前位置的化学场写入 ChemC |

共 2 个腺体通道。

**设计说明：**

- gland_α 对应真实蚂蚁的毒腺/Dufour 腺。释放方式是拖腹——物质沉积在地面，低扩散，形成持久痕迹。
- gland_β 对应真实蚂蚁的大颚腺。释放方式是开颚——物质扩散到空气中，高扩散快衰减，形成短暂的气味云。
- 真实蚂蚁有十几种腺体。两个是精简，但每个的物理行为（地面沉积 vs 空气释放）是诚实的。
- 每个腺体有储量上限和恢复速率。持续分泌会耗尽储量，需要时间补充。这是物理约束，不是游戏机制。

### 4.3 大颚

| 执行器 | 功能 | 信号含义 |
|--------|------|----------|
| mandible | 大颚开合 | 信号高 = 合拢（夹持/切割），信号低 = 张开 |

共 1 个通道。

**设计说明：**

- 大颚合拢在食物上 = 进食（能量回升）。合拢在物体上 = 夹持搬运。合拢在其他蚂蚁上 = 攻击。
- 系统不区分这些"功能"。大颚就是一个物理夹钳。夹住什么东西、产生什么后果，取决于被夹的对象。

### 执行器总计

| 类别 | 数量 |
|------|------|
| 运动 | 3 |
| 腺体 | 2 |
| 大颚 | 1 |
| **合计** | **6** |

## 5. 神经节点类型

沿用线虫版本的六种节点，不新增类型。

| 节点类型 | 功能 |
|----------|------|
| sensor_on | 绑定到一个物理传感器，输出该传感器的分级信号 |
| inter_exc | 兴奋性中间神经元，对输入做加权求和，leaky integrator 时间积分 |
| inter_inh | 抑制性中间神经元，输出抑制目标节点的活动 |
| modulator | 调制神经元，调节其他连接的增益；**新增：驱动可塑突触的权重变化** |
| motor | 绑定到一个物理执行器，驱动该执行器 |

**所有节点保留线虫版的特性：** leaky integrator 时间积分、spike-frequency adaptation、post-inhibitory rebound (PIR)。

## 6. 连接与可塑突触

### 6.1 固定连接（沿用线虫版）

- 玩家手动设定权重 w，运行时不变。
- 正权重 = 兴奋性输入，负权重 = 抑制性输入。
- 权重范围由节点类型决定（Dale's Law）：inter_exc 的输出连接权重 ≥ 0，inter_inh 的输出连接权重 ≤ 0。

### 6.2 可塑连接（蚂蚁版新增）

玩家在接线时可将任意连接标记为"可塑"。可塑连接有以下额外属性：

| 属性 | 说明 |
|------|------|
| w_init | 初始权重，由玩家设定 |
| w | 当前权重，运行时变化，初始值 = w_init |
| w_min, w_max | 权重上下限（clamp），防止无限增长或反转 |
| modulator_binding | 绑定的 modulator 节点（由玩家指定） |
| η (learning_rate) | 学习速率 |
| decay | 权重衰减速率（遗忘），每帧 w 向 w_init 方向缓慢回归 |

### 6.3 权重更新规则

每个仿真步（tick），对每条可塑连接执行：

```
# 学习
pre  = 前端节点的输出（0.0–1.0）
post = 后端节点的输出（0.0–1.0）
mod  = 绑定的 modulator 节点的输出（0.0–1.0）

Δw = η × pre × post × mod

w = w + Δw

# 遗忘
w = w + decay × (w_init - w)

# 限幅
w = clamp(w, w_min, w_max)
```

**设计说明：**

- 规则是标准的调制 Hebbian learning：前后端共同活跃 + 调制信号存在 → 权重增大。三个因子中任何一个为零，都不产生变化。
- 没有"奖励"和"惩罚"的语义区分。modulator 节点只是一个信号源。如果玩家把它接在食物传感器上，吃到食物时权重增大——看起来像"奖励学习"。如果接在伤害传感器上，受伤时权重增大——看起来像"恐惧条件反射"。但这些都是涌现出来的解读，不是系统内置的概念。
- 遗忘（decay toward w_init）确保学到的东西会缓慢消退。如果同一经历反复发生，权重会稳定在一个新值；如果经历停止，权重慢慢回到先天初始值。这和真实突触的自然衰减一致。
- w_min 和 w_max 遵循 Dale's Law。如果前端节点是 inter_exc，权重范围是 [0, w_max]；如果是 inter_inh，范围是 [w_min, 0]。

## 7. 精简决策记录

| 真实蚂蚁硬件 | 决策 | 理由 |
|-------------|------|------|
| 420+ 嗅小球（嗅觉受体类型） | 精简为 4 种化学物种 | 数量精简，但每种的工作方式诚实 |
| 10+ 种外分泌腺 | 精简为 2 个腺体 | 保留了两种物理释放方式（地面沉积 vs 空气挥发） |
| 复眼（复合视觉） | 精简为 1 个明暗通道 | 大多数工蚁视力极差，单通道诚实反映这一事实 |
| 温度/湿度感受器 | 砍掉 | 对核心玩法无贡献，未来可作为解锁传感器加回 |
| 振动感受器（腿部弦音器官） | 砍掉 | 对核心玩法无贡献，未来可作为解锁传感器加回 |
| 本体感受器（Johnston's 器官） | 砍掉 | 触角姿态控制不需要玩家操心 |
| 偏振光导航 | 砍掉 | 二维环境中无天空，不适用 |
| 路径积分（步数计数） | 不内置 | 玩家可用 leaky integrator 反馈环路近似实现，"会漏"是诚实的 |
| 六足步态 | 抽象为转向+前进 | 这是唯一的宏观运动抽象，步态细节对神经回路设计无意义 |
| 蘑菇体 / Kenyon 细胞 | 不内置 | 可塑突触机制提供了相同的物理基础，具体回路结构由玩家搭建 |
| 后退运动 | 砍掉 | 蚂蚁极少后退，精简 |

## 8. 线虫代码封存说明

线虫（nematode）相关代码保留在 `src/creatures/nematode.js`，不删除。蚂蚁实现为 `src/creatures/ant.js`，作为新的默认生物。

线虫与蚂蚁的架构差异：

| 维度 | 线虫 | 蚂蚁 |
|------|------|------|
| 运动模型 | 正弦波蠕动（dorsal/ventral bend） | 转向 + 前进 |
| 传感器布局 | 嘴部六感受器（二维不兼容） | 左右触角（天然二维） |
| 化学输出 | 无 | 2 个腺体向化学场写入 |
| 可塑突触 | 无 | 有（调制 Hebbian learning） |
| 群体交互 | 弱（无社会性） | 强（通过化学场间接通信） |

## 9. 未来扩展预留

以下内容不在当前版本实现，但架构上不应阻碍未来加入：

- **更多化学物种**：ChemE, ChemF...（新增传感器通道 + 新增腺体）
- **更多传感器**：温度、湿度、振动（新增 sensor_on 绑定）
- **蚁后 / 等级分化**：不同个体可以有不同的腺体配置和传感器灵敏度
- **多物种**：不同的 creature 定义文件，不同的身体结构和传感器布局
- **3D 迁移**：运动模型和传感器采样需要扩展到三维，但神经回路层不受影响
