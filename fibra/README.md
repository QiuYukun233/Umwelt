# FIBRA · Braitenberg 沙盒

基于 Braitenberg 车辆理论的交互式神经行为模拟器。拖拽神经连线，观察一只小虫在布满食物与威胁的世界中如何生存、死亡、乃至涌现出四种经典行为。

## 核心概念

[Braitenberg 车辆](https://en.wikipedia.org/wiki/Braitenberg_vehicle) 是一个思想实验：仅凭传感器到电机的直接连线，就能涌现出趋近、回避、追击、依恋等复杂行为。FIBRA 把它做成了可交互沙盒。

**四种基础行为（由连线方式决定）：**

| 行为 | 连线模式 | 现象 |
|------|----------|------|
| 趋近 | 同侧兴奋 | 靠近食物，速度随信号增强 |
| 追击 | 交叉兴奋 | 持续转向食物方向 |
| 依恋 | 同侧抑制 | 在食物旁边减速停留 |
| 回避 | 交叉抑制 | 感知到食物时转向远离 |

每当连线发生变化，程序会自动分析当前回路属于哪一类行为，并在顶栏实时显示。

## 功能一览

- 12 方向双类传感器（食物 × 6、威胁 × 6），可按需启用/关闭，并支持「绝对值」「差分」两种模式
- 拖拽式神经图编辑器：滚轮缩放、拖拽平移、节点「+」按钮、端口拖线、右键删除
- 涌现行为自动分类：趋近 / 追击 / 依恋 / 回避 四种经典模式实时识别
- 完整能量模型：代谢、运动与感知消耗、食物回血、威胁扣血
- 死亡诊断面板：生存时间、食物总数、最终行为、传感器峰值
- 亮/暗主题切换，暂停时仍可编辑回路

## 世界画布

世界是一个可环绕的二维空间，小虫触边后会从对侧出现，轨迹随之重置。

画布内呈现：
- **小虫本体**：朝向由三角形标示，眼睛亮度随传感器信号强度变化
- **12 个传感器锥**：以扇形方式可视化每个启用传感器的感知方向与强度
- **食物与威胁**：食物可被吃掉（原位消失，在地图其它位置重生），威胁碰到扣血
- **淡出轨迹**：记录最近约 100 步的足迹
- **能量环**：包围小虫的环形读数，随剩余量绿 → 橙 → 红渐变

当能量耗尽，会弹出**死亡诊断面板**，显示本轮的生存时间、食物数、最终行为分类、各传感器峰值信号等信息，便于复盘是哪条连线决定了结局。

## 顶栏

- **存活 / 食物 / 代数**：当前轮次的基本统计
- **行为徽章**：实时显示当前电路推断出的行为类别
- **暂停 / 速度**：按钮切换暂停，或 `Space` 键快捷暂停；速度档位 1/8x–2x
- **环境**：面板内调整食物与威胁数量（0 到最大值），点击应用后生效
- **重新投入**：重置小虫位置与能量，代数 +1
- **重置连线**：将神经图恢复为默认状态
- **主题切换**：切换亮/暗外观

## 传感器

小虫身上有 12 个方向传感器，分布在一个圆环上：
- **F0–F5**：食物传感器，按方向排列（正前、右前、右后、正后、左后、左前）
- **T0–T5**：威胁传感器，同样的 6 个方向

点击节点可启用或关闭对应传感器。启用越多，感知越完整，但每秒消耗的能量也越多（激活传感器的信号强度之和 × 0.8 / 秒）。

每个传感器支持两种模式（**右键切换**）：
- **绝对值**：当前方向上的刺激强度
- **差分**：信号变化速率（对运动中的目标更敏感）

## 神经连线编辑器

点击 **「编辑回路」** 进入暂停编辑模式。

### 视图操作

- **滚轮**：缩放画布（0.5–2 倍）
- **拖拽空白处**：平移画布
- **拖拽节点**：重新排布节点位置

### 创建连接

两种方式任选其一：

1. 点击任一输出节点旁的 **「+」按钮**
2. 从输出端口 **拖出一条线**

松开鼠标时根据落点决定后续菜单：

- 落在 **另一节点的输入端口** → 菜单弹出 `⚡ 兴奋` / `⊖ 抑制`，连线直接指向该节点
- 落在 **空白处** → 菜单提供两类选项：
  - `⚡ 兴奋 / ⊖ 抑制 新建节点`：在落点处创建一个中间节点并连线
  - 列出的已有节点：直接连向列表中的某个可输入节点

### 修改与删除

- **点击连线标签**：切换该连线的兴奋/抑制极性
- **右键连线** → 「删除连线」
- **右键中间节点** → 「删除节点」（传感器、电机、内置信号源为系统节点，不可删除）

### 噪声控制

鼠标悬停在 `N_noise` 节点上，节点下方会浮出滑块，调整随机扰动信号的频率（0.1–3 Hz），为行为引入不确定性。

### 内置信号源（始终激活）

- **P_speed**：当前速度的归一化值
- **P_turn**：当前转速的归一化值
- **N_noise**：周期性随机脉冲

这三个源始终存在于神经图中，可作为任意连线的输入。

## 底部状态栏

实时显示各传感器输出、左右腿驱动强度、速度与能量条，以及行为事件日志（食物、威胁、能量告警等）。

## 键盘快捷键

- **Space**：切换暂停/运行（焦点不在输入框或按钮时）

## 能量系统

小虫有 100 点能量，耗尽即死亡：
- 每秒基础衰减 1.5
- 运动消耗：平均电机输出 × 2.0 / 秒
- 传感器消耗：激活传感器的信号强度之和 × 0.8 / 秒
- 吃到食物：+25（食物会在地图其他位置重生）
- 碰到威胁：-18（有 0.45 秒冷却）

能量降到 50% 与 25% 时会在事件日志中给出警告，对应的能量环颜色也会随之变化。

## 项目结构

```
src/
├── main.js          # App 主类，游戏循环、固定时间步长调度
├── config.js        # 全局常量、传感器/连线定义、行为预设
├── world.js         # 世界状态、物理步进、能量/食物/威胁逻辑
├── neural.js        # NeuralGraph：节点/边管理、信号求值、行为推断
├── math.js          # 工具函数（clamp、lerp、角度包裹等）
├── style.css        # 全局样式，支持亮/暗主题
├── renderer/
│   ├── world.js     # Canvas 世界渲染（小虫、食物、威胁、轨迹、传感器锥）
│   ├── graph.js     # 神经图 Canvas 渲染与交互（拖拽、右键菜单）
│   └── sensor-map.js # 传感器环形图渲染
└── ui/
    ├── topbar.js    # 顶栏 DOM 操作（统计、速度、主题、空格键暂停）
    ├── footer.js    # 底部数据栏与事件日志
    ├── sidebar.js   # 侧边栏容器，整合传感器图和神经编辑器
    ├── editor.js    # 神经编辑器面板（噪声控制、编辑/运行按钮）
    └── death.js     # 死亡诊断弹窗
```

## 技术栈

- **Vite 7** 构建与开发服务器，无其它运行时依赖
- 纯原生 JavaScript（ES6 模块）+ Canvas 2D API
- 固定 60Hz 时间步长的物理模拟，通过累加器循环与渲染帧解耦
